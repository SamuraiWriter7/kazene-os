# Relationship to KazeneOS

## Overview

This repository defines **Royalty OS** as a value-circulation sub-OS inside **KazeneOS**.

Royalty OS is not a separate civilizational theory competing with KazeneOS.  
It is a specialized implementation layer within KazeneOS, focused on permission, trace, and allocation.

In short:

> **KazeneOS is the upper-layer OS.**  
> **Royalty OS is its value-circulation sub-OS.**

---

## One-Sentence Definition

> **KazeneOS integrates question, trace, resonance, circulation, and governance at the civilizational level, while Royalty OS implements value return through Permission, Trace, and Allocation inside that structure.**

---

## Why Royalty OS exists

KazeneOS defines the broader structure of the AI era:

- question
- trace
- resonance
- circulation
- responsibility
- governance

However, value circulation requires a more concrete operational layer.

Royalty OS exists to answer practical questions such as:

- What may be used?
- What was actually used?
- What influenced the output?
- Who should receive return?
- How should the return process be audited?

This is the role of Royalty OS.

---

## Division of Responsibility

## KazeneOS handles:
- civilizational meaning
- the philosophy of trace
- the reason circulation is necessary
- the relation between response, responsibility, and return
- upper-layer governance logic

## Royalty OS handles:
- permission conditions
- usage records
- traceable evidence
- allocation logic
- operational return
- auditable circulation

This means:

- **KazeneOS defines the meaning of circulation**
- **Royalty OS defines the procedure of circulation**

---

## Internal Structure of Royalty OS

Royalty OS is built around three core layers:

### Permission
Defines what may be used and under what conditions.

### Trace
Records what was used, when, and how.

### Allocation
Determines how recorded value should be distributed or returned.

These three layers form a closed loop:

```text
Permission → Trace → Allocation

This loop is the operational core of Royalty OS.

Trace as the Main Interface

The most important connection between KazeneOS and Royalty OS is Trace.

In KazeneOS, trace is not merely a log.
It is the structural residue between question and response.

In Royalty OS, trace is not merely a memory.
It is returnable evidence used for allocation and audit.

Therefore, trace is the main interface that links the philosophical layer of KazeneOS with the operational layer of Royalty OS.

At minimum, trace can be divided into:

Access Trace — what was accessed
Influence Trace — what affected the output
Audit Trace — who guarantees the record

Through this structure:

KazeneOS provides the theory of trace
Royalty OS provides the implementation of trace-based return
Structural Placement
KazeneOS
└─ Value Circulation Layer
   └─ Royalty OS
      ├─ Permission
      ├─ Trace
      └─ Allocation

Royalty OS should therefore be understood as:

not outside KazeneOS
not above KazeneOS
not parallel in a competing sense

but rather:

inside KazeneOS
below its upper principles
specialized for value circulation
Why the Distinction Matters

If Royalty OS is separated too far from KazeneOS, it risks shrinking into a mere copyright-processing system.
If KazeneOS absorbs Royalty OS without distinction, practical implementation becomes blurred.

The optimal structure is:

separate names
unified architecture
clear hierarchy

This preserves both:

the philosophical breadth of KazeneOS
the operational specificity of Royalty OS
Design Principle

Do not merge the names too early.
Unify the structure instead.

KazeneOS should remain the upper-layer framework.
Royalty OS should remain a clearly named subsystem for value return.

This makes the architecture easier to explain, implement, and expand.

Short Summary

Royalty OS is the value-circulation subsystem of KazeneOS.
KazeneOS defines the upper civilizational logic of question, trace, resonance, circulation, and governance.
Royalty OS implements that logic through Permission, Trace, and Allocation.

In this sense:

KazeneOS is the upper constitution.
Royalty OS is the circulation kernel inside it.
